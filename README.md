# Тестовый проект по автоматизации тестирования UI для мобильного приложения

Данный проект представляет собой **фреймворк для автоматизации тестирования мобильных приложений** (Android), разработанный в рамках тестового задания. Он демонстрирует построение масштабируемой и гибкой инфраструктуры для UI-тестов с использованием современных инструментов.

## Основные возможности

- Поддержка **нескольких APK** (приложения условно названы `vkvideo` и `alchemy`) с переключением через аннотации.
- Запуск тестов на **локальном эмуляторе** или в **Selenoid** (контейнерный запуск через docker-compose), настраивается через параметр `test.run.type`.
- Динамическая загрузка конфигураций из `.properties`-файлов в зависимости от выбранного приложения и типа запуска.
- Интеграция с **Allure** для детальных отчётов: скриншоты при падениях, шаги, кастомные метки.
- Автоматическая обработка всплывающих виджетов (например, `Sign In`) через JUnit-extensions.
- Управление сетевыми настройками (отключение интернета, Wi-Fi, мобильных данных) с сохранением исходного состояния и восстановлением после теста.

## Стек

- **Java 22**
- **Maven 3.9.8**
- **JUnit 5**
- **Selenide 7** 
- **Appium** (java-client 10.x)
- **Allure**
- **Android SDK** (эмуляторы API 30+)

## Предварительные требования
- Установленный **Android SDK** с `platform-tools` и `build-tools` (например, версии 30.0.3).
- **Appium server** (рекомендуется версия 3+), запущенный с необходимыми флагами:
- Настроенный эмулятор (Android Studio, либо плагин Android)
  
  ```bash
  appium --allow-insecure='*:adb_shell,*:session_discovery'
  ```
  
## Запуск и получение отчета
- Через maven
  
  ```bash
  mvn clean test -Dtest.run.type=local
  ```

- Получение allure отчета
  
  ```bash
  mvn allure:serve
  ```
